## 專案概述

**12 Week Year App** 是一個生產力應用程式，將傳統年度目標壓縮為 12 週執行週期，幫助使用者高效達成目標。

### 核心功能模組

```
┌─────────────────────────────────────────────────────────────┐
│                    12 Week Year App                          │
├──────────────┬──────────────┬──────────────┬────────────────┤
│   願景設定    │   12週計劃   │   每週計劃    │    計分卡      │
│   Vision     │    Goals     │  Weekly Plan  │   Scorecard   │
├──────────────┴──────────────┴──────────────┴────────────────┤
│                      問責會議 (WAM)                          │
│                    Accountability                            │
├─────────────────────────────────────────────────────────────┤
│   行事曆整合  │   推播通知   │   第13週回顧  │    匯出功能    │
│   Calendar   │    Push     │   Week 13    │    Export     │
└─────────────────────────────────────────────────────────────┘
```

## 前後端任務依賴關係

開發時需注意以下依賴順序，後端 API 需先完成才能進行對應前端整合：

```
┌─────────────────────────────────────────────────────────────────┐
│                        Phase 1 (並行開發)                        │
├────────────────────────────┬────────────────────────────────────┤
│ 前端                        │ 後端                               │
│ • Monorepo 設置             │ • Hono 專案初始化                   │
│ • UI 元件庫整合              │ • D1 Schema 設計                   │
│ • 狀態管理配置               │ • 中介軟體開發                      │
└────────────────────────────┴────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        Phase 2 (依賴開發)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [後端] Auth API ──────────────→ [前端] 認證模組 UI              │
│         ↓                                                       │
│  [後端] Vision API ────────────→ [前端] 願景設定頁面             │
│         ↓                                                       │
│  [後端] Goals API ─────────────→ [前端] 目標管理頁面             │
│         ↓                                                       │
│  [後端] Tactics API ───────────→ [前端] 戰術管理頁面             │
│         ↓                                                       │
│  [後端] Weekly API ────────────→ [前端] 每週計劃頁面             │
│         ↓                                                       │
│  [後端] Scorecard API ─────────→ [前端] 計分卡頁面               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        Phase 3 (依賴開發)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [後端] Accountability API ───→ [前端] 問責夥伴 UI              │
│  [後端] Calendar API ─────────→ [前端] 行事曆整合                │
│  [後端] Notification Service ─→ [前端] 推播通知 UI              │
│  [後端] Export API ───────────→ [前端] 匯出功能                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 關鍵依賴節點

| 後端任務 | 前端阻塞任務 | 建議緩衝 |
|----------|--------------|----------|
| Auth API | 所有需認證頁面 | 優先完成 |
| Goals API | 目標、戰術、計分卡 | 2 天 |
| Weekly API | 每週計劃、計分卡 | 2 天 |
| Notification Service | 推播設定 UI | 3 天 |

### 建議開發策略

1. **Mock API 先行**：前端可先用 MSW 模擬 API 開發 UI
2. **型別共享**：packages/shared 定義後立即同步
3. **API 契約優先**：先定義 OpenAPI spec 再實作

## 技術棧摘要

### 前端

| 平台 | 技術 |
|------|------|
| Web | Next.js 15 + React 19 + TailwindCSS |
| Mobile | React Native 0.76 + Expo SDK 52 |
| 狀態管理 | Zustand 5.x + TanStack Query 5.x |
| UI 元件 | Radix UI (Web) / React Native Paper (Mobile) |

### 後端

| 類別 | 技術 |
|------|------|
| 框架 | Hono 4.x |
| Runtime | Cloudflare Workers |
| 資料庫 | Cloudflare D1 (SQLite) |
| 儲存 | Cloudflare R2 + KV |
| 認證 | JWT (jose 5.x) |

## 優先級定義

| 等級 | 定義 | 說明 |
|------|------|------|
| P0 | Critical | MVP 必要功能，上線阻擋項 |
| P1 | High | 重要功能，Beta 版本必須 |
| P2 | Medium | 加分功能，提升體驗 |
| P3 | Low | 未來迭代，可延後 |

1. **使用 GitHub Issues** 追蹤每個任務

3. **每週 Review** 更新任務狀態
4. **標記依賴關係** 確保開發順序
